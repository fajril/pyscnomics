Chapter 1: Fundamentals of Petroleum Project Economics
======================================================

Before diving into Production Sharing Contracts (PSC), it is essential to
understand the basics of petroleum project economics. This chapter explains
how to evaluate oil and gas projects using standard economic principles,
without any fiscal regime complexities.

1.1 Introduction to Project Economics
--------------------------------------

Petroleum project economics is the discipline of evaluating the financial
viability of oil and gas investments. Unlike PSC-based analysis, which involves
revenue sharing between contractors and governments, basic project economics
focuses on the project's standalone profitability.

**Key Components of Project Economics:**

1. **Revenue** - Income from selling hydrocarbons
2. **Costs** - Capital and operating expenditures
3. **Taxes** - Corporate income taxes and indirect taxes
4. **Cash Flow** - Net cash generated by the project
5. **Economic Indicators** - NPV, IRR, Payout Time

**When to Use Basic Project Economics:**

- Early-stage prospect evaluation
- Comparing projects across different fiscal regimes
- Corporate portfolio analysis
- Understanding baseline project value before fiscal terms

1.2 The BaseProject Class
-------------------------

The ``BaseProject`` class is the foundation of all economic calculations in
pyscnomics. It represents a standard oil and gas project without any PSC
mechanisms.

**Inheritance Hierarchy:**

.. code-block:: text

   BaseProject (standard project economics)
       ├── CostRecovery (adds PSC cost recovery)
       └── GrossSplit (adds PSC gross split)

1.2.1 Core Attributes
^^^^^^^^^^^^^^^^^^^^^

A BaseProject requires these essential inputs:

**Project Timeline:**

- ``start_date``: Project commencement date
- ``end_date``: Project completion date
- ``oil_onstream_date``: Date when oil production begins
- ``gas_onstream_date``: Date when gas production begins

**Production Data:**

- ``lifting``: Production volumes and prices (oil, gas, etc.)

**Cost Categories:**

- ``capital_cost``: Tangible investments (facilities, wells, equipment)
- ``intangible_cost``: Non-depreciable costs (licenses, studies)
- ``opex``: Operating expenditures (maintenance, personnel)
- ``asr_cost``: Abandonment and Site Restoration costs
- ``lbt_cost``: Land and Building Tax
- ``cost_of_sales``: Transportation and marketing costs

1.2.2 Project Economics Calculation Flow
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

The BaseProject.run() method executes the following calculation sequence:

**Step 1: Revenue Calculation**

- Calculate production volumes (lifting)
- Apply prices to get revenue by fluid type
- Handle multiple revenue streams (oil, gas, sulfur, electricity, CO2)

**Step 2: Cost Preparation**

- Validate cost objects
- Calculate sunk costs (costs incurred before project start)
- Calculate pre-onstream costs (costs before production begins)

**Step 3: Expenditure Calculation**

- Calculate pre-tax expenditures
- Apply inflation adjustments
- Calculate indirect taxes (VAT, import duties)
- Calculate post-tax expenditures

**Step 4: Cash Flow Calculation**

- Cash Flow = Revenue - Sunk Costs - Pre-onstream Costs - Post-tax Expenditures

**Step 5: Consolidation**

- Combine oil and gas profiles
- Generate consolidated project metrics

1.3 Revenue Calculation
-----------------------

Revenue is the product of production volume and price:

.. math::

   R_t = \sum_{f} Q_{f,t} \times P_{f,t}

Where:

- :math:`R_t` = Total revenue in year t (see Chapter 0 notation)
- :math:`Q_{f,t}` = Production volume of fluid f in year t
- :math:`P_{f,t}` = Price of fluid f in year t
- :math:`f` = Fluid type (oil, gas, sulfur, electricity, CO2)

.. note::
   For detailed notation conventions, see :doc:`chapter0_math_notation`.

**Supported Revenue Streams:**

+-------------------+-----------------------------------+
| Fluid Type        | Description                       |
+===================+===================================+
| Oil               | Crude oil production              |
+-------------------+-----------------------------------+
| Gas               | Natural gas production            |
+-------------------+-----------------------------------+
| Sulfur            | Sulfur byproduct                  |
+-------------------+-----------------------------------+
| Electricity       | Power generation                  |
+-------------------+-----------------------------------+
| CO2               | Carbon capture/utilization        |
+-------------------+-----------------------------------+

**Weighted Average Price (WAP):**

When multiple lifting transactions occur at different prices in the same year,
pyscnomics calculates a volume-weighted average price to ensure accurate
revenue calculations.

1.4 Cost Categories
-------------------

Understanding cost classification is crucial for accurate project economics.

1.4.1 Capital Costs (Tangible)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

Capital costs represent investments in physical assets with multi-year useful
lives. These costs are depreciated over time.

**Characteristics:**

- Long-term assets (facilities, wells, platforms, pipelines)
- Subject to depreciation
- Often large upfront investments
- May qualify for tax incentives

**In pyscnomics:**

.. code-block:: python

   from pyscnomics.econ.costs import CapitalCost
   from pyscnomics.econ.selection import FluidType

   capital_cost = CapitalCost(
       start_year=2020,
       end_year=2030,
       cost=np.array([50_000_000, 30_000_000]),  # Year 2020, 2021
       expense_year=np.array([2020, 2021]),
       cost_allocation=[FluidType.OIL, FluidType.OIL],
       is_ic_applied=True,  # Investment Credit applicable
   )

1.4.2 Intangible Costs
^^^^^^^^^^^^^^^^^^^^^^

Intangible costs are expenses that don't result in physical assets but are
necessary for project development.

**Characteristics:**

- Non-physical expenditures (studies, licenses, permits)
- Expensed immediately or amortized
- Often incurred early in project life
- Not depreciable

**Examples:**

- Geological and geophysical studies
- Environmental impact assessments
- Legal fees
- Training costs

1.4.3 Operating Expenditure (OPEX)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

OPEX represents ongoing costs to maintain production.

**Characteristics:**

- Recurring annual costs
- Varies with production level
- Includes maintenance, personnel, utilities

**Calculation:**

.. math::

   \text{OPEX}_t = \text{Fixed Cost}_t + \text{Variable Cost}_t \times Q_t

Where:

- :math:`\text{Fixed Cost}_t` = Base operating cost regardless of production
- :math:`\text{Variable Cost}_t` = Cost per unit of production
- :math:`Q_t` = Production in year t

.. note::
   All monetary values in this chapter follow the MUSD (thousand USD) convention from :doc:`chapter0_math_notation`.

1.4.4 Abandonment and Site Restoration (ASR)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

ASR costs represent obligations to restore the site after production ends.

**Characteristics:**

- Incurred at end of project life
- May be funded through annual provisions
- Required by environmental regulations

1.4.5 Land and Building Tax (LBT)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

LBT represents annual taxes on land and buildings used in the project.

**Characteristics:**

- Annual recurring cost
- Based on property value
- Varies by jurisdiction

1.4.6 Cost of Sales
^^^^^^^^^^^^^^^^^^

Cost of sales includes transportation and marketing costs to deliver products
to market.

**Characteristics:**

- Varies with production volume
- Includes shipping, pipeline tariffs, marketing fees

1.5 Expenditure Calculation
---------------------------

BaseProject calculates expenditures through multiple stages:

1.5.1 Pre-Tax Expenditures
^^^^^^^^^^^^^^^^^^^^^^^^^^

Raw costs before any tax adjustments:

.. math::

   E_{pre-tax} = \text{Capital} + \text{Intangible} + \text{OPEX} + \text{ASR} + \text{LBT} + \text{Cost of Sales}

1.5.2 Inflation Adjustment
^^^^^^^^^^^^^^^^^^^^^^^^^^

Costs can be adjusted for inflation:

.. math::

   E_{inflated} = E_{pre-tax} \times (1 + i)^t

Where:
- :math:`i` = Inflation rate
- :math:`t` = Number of years from base year

1.5.3 Indirect Taxes
^^^^^^^^^^^^^^^^^^^^

Indirect taxes (VAT, import duties) are calculated on applicable expenditures:

.. math::

   \text{Tax}_{indirect} = E_{taxable} \times \tau_{indirect}

1.5.4 Post-Tax Expenditures
^^^^^^^^^^^^^^^^^^^^^^^^^^^

Final expenditures after adding indirect taxes:

.. math::

   E_{post-tax} = E_{pre-tax} + \text{Tax}_{indirect}

1.6 Cash Flow Calculation
-------------------------

The fundamental cash flow equation for a base project:

.. math::

   CF_t = R_t - SC_t - POC_t - E_{post-tax,t}

Where:

- :math:`CF_t` = Cash flow in year t
- :math:`R_t` = Revenue in year t
- :math:`SC_t` = Sunk costs in year t (costs before project start)
- :math:`POC_t` = Pre-onstream costs in year t (costs before production)
- :math:`E_{post-tax,t}` = Post-tax expenditures in year t

**Cash Flow Components:**

1. **Positive Cash Flow** (typically during production):

   - Revenue from hydrocarbon sales
   - Other revenue streams

2. **Negative Cash Flow** (typically early years):

   - Capital investments
   - Exploration costs
   - Development drilling

3. **Operating Cash Flow** (during production):

   - Revenue minus operating costs
   - Taxes

1.7 Using BaseProject in pyscnomics
-----------------------------------

Here's a complete example of using BaseProject:

.. code-block:: python

   import numpy as np
   from datetime import date
   from pyscnomics.contracts import BaseProject
   from pyscnomics.econ.costs import CapitalCost, OPEX
   from pyscnomics.econ.revenue import Lifting
   from pyscnomics.econ.selection import FluidType
   from pyscnomics.tools.table import get_table

   # Define project timeline
   project = BaseProject(
       start_date=date(2020, 1, 1),
       end_date=date(2035, 12, 31),
       oil_onstream_date=date(2022, 1, 1),
   )

   # Define production profile
   oil_production = Lifting(
       start_year=2020,
       end_year=2035,
       prod_year=np.arange(2020, 2036),
       lifting_rate=np.array([0, 0, 5000, 8000, 10000, 10000, 9000, 
                              8000, 7000, 6000, 5000, 4000, 3000, 
                              2000, 1000, 500]),
       price=np.array([60.0] * 16),
       fluid_type=FluidType.OIL,
   )

   # Define capital costs
   capex = CapitalCost(
       start_year=2020,
       end_year=2035,
       expense_year=np.array([2020, 2021, 2022]),
       cost=np.array([100_000_000, 150_000_000, 50_000_000]),
       cost_allocation=[FluidType.OIL] * 3,
   )

   # Define OPEX
   opex = OPEX(
       start_year=2020,
       end_year=2035,
       expense_year=np.arange(2022, 2036),
       fixed_cost=np.array([5_000_000] * 14),
       cost_allocation=[FluidType.OIL] * 14,
   )

   # Assign to project
   project.lifting = (oil_production,)
   project.capital_cost = (capex,)
   project.opex = (opex,)

   # Run calculation
   project.run(
       tax_rate=0.10,  # 10% indirect tax
       inflation_rate=0.02,  # 2% annual inflation
   )

   # Access results using get_table
   table_oil, table_gas, table_consolidated = get_table(project)

   # Access results from DataFrame
   print(f"Total Revenue: ${table_consolidated['Revenue'].sum():,.0f}")
   print(f"Total Expenditures: ${table_consolidated['Total_Expenditures_Post_Tax'].sum():,.0f}")
   print(f"Net Cash Flow: ${table_consolidated['Cashflow'].sum():,.0f}")

1.8 Economic Indicators
-----------------------

BaseProject results can be used to calculate standard economic indicators:

**Net Present Value (NPV):**

.. math::

   NPV = \sum_{t=0}^{T} \frac{CF_t}{(1 + r)^t}

Where:
- :math:`r` = Discount rate (e.g., 10%)
- :math:`T` = Project life

**Internal Rate of Return (IRR):**

The discount rate at which NPV equals zero:

.. math::

   \sum_{t=0}^{T} \frac{CF_t}{(1 + IRR)^t} = 0

**Payout Time:**

The time required for cumulative cash flows to turn positive.

1.9 Transition to PSC Analysis
------------------------------

Once you understand basic project economics, you can analyze how different
fiscal regimes affect project value:

- **Cost Recovery PSC**: Adds government take through FTP, cost recovery caps,
  and profit sharing (see :doc:`chapter6_cost_recovery`)

- **Gross Split PSC**: Adds government take through direct revenue splitting
  (see :doc:`chapter7_gross_split`)

The key insight is that PSC mechanisms modify the cash flow distribution
between contractor and government, but the underlying project economics
(revenue and costs) remain the same.

1.10 Summary
------------

Key takeaways from this chapter:

1. **BaseProject** provides the foundation for all economic calculations
2. **Revenue** comes from hydrocarbon production at market prices
3. **Costs** include capital, intangible, operating, and other expenses
4. **Cash Flow** = Revenue - All Costs (including taxes)
5. **Economic Indicators** (NPV, IRR) measure project profitability
6. Understanding base economics is essential before analyzing PSC impacts
